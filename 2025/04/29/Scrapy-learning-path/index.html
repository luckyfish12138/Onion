爬虫Scrapy基础

第一章 爬虫的基本介绍

1.网络请求

http://www.baidu.com —— url 统一资源定位符

请求过程：

客户端，指web浏览器向服务器发送请求

请求分为四部分：

1.请求网址 – request url

2.请求方法 – request methods

3.请求头 – request header

4.请求体 – request body

在浏览器中摁f12 查看请求响应

2.爬虫的概念

模拟阅览器、发送请求、获取响应

只能获取客户端所展示的内容

特点：知识碎片化

在写爬虫的时候会面对各种各样的网站，每个网站都是有区别的

模拟客户端，操作者是正常的用户

作为一个爬虫，服务器不欢迎我们的，尽量去模拟正常的用户发送请求

爬虫 — 模拟客户端访问，抓取数据

反扒 — 保护重要数据，阻止恶意网络攻击

反反爬 — 针对反扒做出的措施

2.1 爬虫的作用

1.数据采集

2.软件测试

3.抢票

4.网络安全

5.web漏洞扫描

2.2 爬虫分类

根据爬取网站的数量：

1.通用爬虫

特点：爬取网站数量没有上限

示例：搜索引擎

2.聚焦爬虫（重点）

特点：爬取网站数量有上限，有明确目标

分类:

1.功能性爬虫：

特点：不获取数据，只为实现某一功能

示例：抢票/短信轰炸

2.数据增量爬虫：

特点：获取时间用于后续分析

分类：

url与数据同时变化：整条数据更新

url不变，数据裱花：数据部分更新

3.爬虫的基本流程

3.1 基本流程

流程：url(网址资源定位符) ——> 对url发送网络请求，获取网络请求的响应——>解析响应，提前数据——> 保存数据

1.确认目标：目标url：www.baidu.com

2.发送请求 ：发送网络请求，获取到特定的服务端给你的响应

3.提前数据：从响应中提取特定的数据 jsonpath/xpath/re

4.保存数据：本地（html、json、txt）、数据库

3.2 robots协议

robots 协议并不是一个规范，只是约定俗成

3.3 请求头

1.网络通信步骤：

电脑（阅览器）：url – www.baidu.com

DNS服务器：ip地址标注服务器 – 1.1.1.1 IP地址

DNS服务器返回ip地址给阅览器

阅览器拿到ip地址去访问阅览器，返回响应

服务器返回给我们的响应数据，html/css/js/jpg

百度首页：时间上由很多部分组成

html：文本

css：样式，控制文字大小、颜色

js：行为，包括鼠标点击

jpg：图片

网络通信的实际原理：

一个请求只能对应一个数据包（文件）

3.4 http协议 和 https协议

http协议 ：规定了服务器和客户端相互通信的规则，超文本传输协议，默认端口号是80

超文本：不仅仅限于文本，还包括图片，音频，视频

传输协议指使用共用的约定格式来传递转换成字符串的超文本内容

https 协议：http + ssl （安全套接字层）默认端口号是443

带有安全套接字层的超文本传输协议

ssl对传输的内容进行加密

https比http更安全，但是性能更低

http请求/响应的步骤：

1.客户端连接到web服务器

2.发送http请求

3.服务器接受请求返回响应

4.释放连接tcp连接

5.客户解析html内容

3.5 请求头格式

请求头请求方式：

get 向服务器要资源

post 向服务器提交资源

user-agent: 模拟正常用户

cookie：登录保持

referer：当前这一次请求是由那个请求来的

抓包得到的响应内容才是判断依据，elements中的代码是渲染之后的代码不能作为判断标准

3.6 requests 模块

requests 模块作用：发送http请求，获取响应数据

requests 模块是第三方模块，需要安装
